# 华电交友（friend-app）

> 面向大学生/研究生的校园交友产品 —— **网页版可开发原型需求文档（V1）**

---

## 1. 文档目标与本期范围

本文档用于把“产品想法”落到“可直接开发的前端原型需求”。

### 1.1 本期目标
- 产出一个可演示、可点击、可走通主流程的 Web 原型。
- 验证“真实身份 + 同频匹配 + 轻线下活动”的交友模式。
- 便于后续对接后端与算法团队。

### 1.2 网页版原型开发范围（严格限定）
**本阶段只做 React + Vite 前端静态原型，不接真实后端，全部使用 Mock 数据。**

- 技术栈：React + Vite + TypeScript（建议）+ React Router
- 数据来源：本地 `mock/*.ts` / `mock/*.json`
- 数据行为：页面内存态/本地状态管理（Context 或 Zustand 均可）
- 不包含：
  - 真实登录鉴权
  - 数据库存储
  - 实时 WebSocket 消息
  - 真实推荐算法服务

### 1.3 成果定义（DoD）
- 至少 7 个核心页面可路由跳转。
- 每页有明确组件拆分和交互说明。
- Mock 数据可覆盖完整主流程：注册→完善资料→匹配推荐→聊天→活动报名→安全反馈。

## 2. 产品定位（简版）

**产品名：华电交友**

**核心人群：** 华北电力大学及周边高校本科生/研究生。

**价值主张：**
- 真实：校园认证与信用体系
- 同频：关系意图 + 时间匹配 + 兴趣标签
- 安全：隐私保护、举报拉黑、线下见面提示

## 3. 信息架构与页面结构

本期原型至少包含以下页面：

1. 登录/注册页（`/auth`）
2. 首页推荐页（`/home`）
3. 个人资料页（`/profile`）
4. 匹配推荐页（`/match`）
5. 聊天页（`/chat`）
6. 活动广场页（`/events`）
7. 安全中心页（`/safety`）

### 3.1 建议导航结构
- 顶部：Logo + 校园认证状态 + 全局搜索（可选）
- 底部/侧边主导航：首页、匹配、聊天、活动、我的
- 安全中心可从“我的”与聊天页快捷入口进入

## 4. 页面级需求（组件 + 交互）

## 4.1 登录/注册页 `/auth`
**目标：** 完成用户进入产品的最小流程。

**核心组件：**
- 登录 Tab / 注册 Tab 切换
- 输入框：手机号/邮箱、密码、验证码（模拟）
- 学校选择器（华北电力大学默认）
- 用户协议与隐私政策勾选
- CTA 按钮：登录 / 注册

**关键交互：**
- 注册成功后跳转资料完善弹窗（或 `/profile` 首次引导）。
- 对未勾选协议进行前端校验提示。
- 校园认证状态先显示“未认证（Mock）”。

## 4.2 首页推荐页 `/home`
**目标：** 低压力浏览高质量推荐对象。

**核心组件：**
- 顶部欢迎区（今日推荐数）
- 推荐卡片流（头像、昵称、学院、关系意图、兴趣）
- 推荐理由标签（如“作息接近”“共同兴趣3项”）
- 操作按钮：感兴趣/跳过/稍后再看

**关键交互：**
- 点击卡片进入对方详情抽屉（不离开当前页）。
- “感兴趣”后写入本地状态，并在匹配页可见。
- 每日推荐上限提示（Mock 逻辑）。

## 4.3 个人资料页 `/profile`
**目标：** 完善个人信息，提高匹配质量。

**核心组件：**
- 头像与基础信息区（昵称、性别、年级、学院）
- 关系意图单选组件
- 兴趣标签多选组件
- 作息与空闲时间段选择器
- 自我介绍文本框
- 认证区：校园邮箱/学号（仅展示 Mock 认证流程）
- 信用分展示卡

**关键交互：**
- 资料完整度进度条（例如 0–100%）。
- 保存后 Toast 提示；完整度提升影响推荐权重（前端展示）。
- 认证提交后状态变为“审核中（Mock）/已认证（Mock）”。

## 4.4 匹配推荐页 `/match`
**目标：** 展示“同频匹配”结果与可解释性。

**核心组件：**
- 匹配筛选栏（关系意图、学院、在线时间）
- 匹配列表卡片（匹配分数、共同点、距离/校区）
- 匹配解释模块（分数构成：意图/兴趣/时间/信用）
- 快捷操作：发起聊天、收藏、加入活动

**关键交互：**
- 调整筛选条件实时刷新列表（本地过滤）。
- 点击“查看分数”展开分数详情。
- 支持“仅同校认证用户”开关。

## 4.5 聊天页 `/chat`
**目标：** 验证从匹配到沟通的核心链路。

**核心组件：**
- 会话列表（最近消息、未读数）
- 聊天窗口（消息气泡、时间戳、已读状态 Mock）
- 破冰话题快捷按钮
- 安全工具条：举报/拉黑/安全提示

**关键交互：**
- 左侧选中会话，右侧加载对应消息。
- 点击破冰话题可自动填入输入框。
- 举报/拉黑弹出二次确认对话框。

## 4.6 活动广场页 `/events`
**目标：** 促进线下轻社交，减少“只聊不见”。

**核心组件：**
- 活动分类筛选（学习、运动、桌游、散步）
- 活动卡片（时间、地点、人数、发起人认证状态）
- 活动详情弹窗
- 报名按钮与报名状态

**关键交互：**
- 报名后人数实时变化（Mock）。
- 活动结束后可触发“互评入口（Mock）”。
- 对低信用用户显示报名限制提示（Mock 规则）。

## 4.7 安全中心页 `/safety`
**目标：** 建立用户安全感与平台边界。

**核心组件：**
- 账号与隐私设置区（可见范围、消息权限）
- 举报记录列表
- 拉黑名单管理
- 线下见面安全指南卡片
- 常见风险提示（冒充身份、引流诈骗等）

**关键交互：**
- 可切换“仅同校可见”隐私选项。
- 举报后生成记录并显示处理状态（Mock）。
- 拉黑后聊天列表即时隐藏对方（前端状态）。

## 5. 视觉与体验规范（Web 原型）

### 5.1 视觉风格关键词
- **简洁、ins 风格、清爽留白、轻社交氛围**
- 强调“真实、同频、可靠”，避免“颜值评分”导向

### 5.2 UI 要求
- 主色建议：低饱和暖色 + 中性灰
- 字体层级清晰：标题/正文/辅助说明
- 卡片圆角统一，阴影轻量
- 动效轻微（150–250ms），不花哨

### 5.3 内容安全与价值导向
- 不使用“擦边”“低俗”文案与视觉暗示
- 不提供公开颜值打分机制
- 推荐理由优先展示：兴趣、时间、目标一致性

## 6. Mock 数据设计

为保证原型可跑通，至少准备以下 Mock 数据结构：

### 6.1 用户 `mockUsers`
字段建议：
- `id`, `nickname`, `gender`, `age`, `school`, `college`, `grade`
- `bio`, `avatar`, `verifiedCampus`（bool）
- `intent`（恋爱/学习搭子/活动玩伴/长期朋友）
- `interests`（string[]）
- `availableSlots`（string[]）
- `creditScore`（0-100）

### 6.2 推荐对象 `mockRecommendations`
- `userId`, `reasonTags`, `matchScore`, `distanceHint`, `updatedAt`

### 6.3 聊天记录 `mockChats`
- `conversationId`, `peerUserId`, `messages[]`
- `messages` 包含：`senderId`, `content`, `timestamp`, `status`

### 6.4 校园活动 `mockEvents`
- `eventId`, `title`, `category`, `hostUserId`
- `startTime`, `location`, `capacity`, `joinedCount`, `requireVerified`

### 6.5 匹配分数 `mockMatchBreakdown`
- `userId`, `targetUserId`
- `intentScore`, `interestScore`, `timeScore`, `creditScorePart`, `finalScore`

### 6.6 信用分 `mockCreditLogs`
- `userId`, `change`, `reason`, `createdAt`

### 6.7 兴趣标签 `mockInterestTags`
- 学习类、运动类、艺术类、游戏类、社交类等枚举

## 7. 产品安全边界（必须在原型中体现）

### 7.1 校园认证
- 首页/资料页明确展示认证状态：未认证、审核中、已认证（Mock）
- 可配置“仅向已认证用户展示我”

### 7.2 举报与拉黑
- 聊天页与个人主页都要提供举报入口
- 举报需选择原因（骚扰、诈骗引流、冒充身份等）
- 拉黑后立即生效：不可再发消息、不可被推荐

### 7.3 隐私保护
- 真实姓名、学号、联系方式默认不公开
- 个人资料可配置可见范围（全部、同校、匹配后可见）

### 7.4 线下见面安全提示
- 首次线下见面前弹出提示卡：公共场所、告知朋友、夜间谨慎
- 可一键复制“行程报备模板”（前端 Mock 功能）

## 8. 推荐逻辑（前端演示版）

本期仅做可解释展示，不接真实算法服务。

建议展示公式：

`MatchScore = 35%关系意图 + 25%兴趣重合 + 20%时间重合 + 10%地理便利 + 10%互动质量`

页面中用进度条/雷达图展示分项得分，帮助用户理解“为什么推荐这个人”。

## 12. 安装与运行（React + Vite 原型）

### 环境要求
- Node.js 18+
- npm 9+

### 安装依赖
```bash
npm install
```

### 启动开发环境
```bash
npm run dev
```

### 默认访问
- Vite 启动后打开终端提示的本地地址（通常为 `http://localhost:5173`）
- 首次进入默认路由会跳转到 `/auth`

### 构建产物
```bash
npm run build
npm run preview
```
